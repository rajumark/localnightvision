Local Night Vision

On-device low-light image enhancement for Android & iOS

Kotlin Multiplatform Compose Multiplatform TensorFlow Lite MIRNet CameraX iOS (WIP)

Overview

Local Night Vision brings MIRNet — a state-of-the-art deep learning architecture for low-light image enhancement — directly to mobile devices. All processing happens on-device via TensorFlow Lite with GPU acceleration. No data ever leaves your phone.

The app provides a live camera viewfinder, gallery image picker, and a touch-to-compare result viewer that lets you hold to see the original versus the enhanced version.

Research Paper

MIRNet: Learning Enriched Features for Real Image Restoration and Enhancement
Syed Waqas Zamir, Aditya Arora, Salman Khan, Munawar Hayat, Fahad Shahbaz Khan, Ming-Hsuan Yang, Ling Shao
arXiv:2003.06792v2

MIRNet introduces a multi-scale residual block that maintains spatially-precise high-resolution representations while receiving rich contextual information from low-resolutions. The architecture achieves state-of-the-art results on low-light enhancement, image denoising, and super-resolution.

Data Flow

Camera / Gallery → ByteArray → Decode + Resize (400×400) → Normalize [0,1] → TFLite Inference → Denormalize → Result JPEG

Platform Modules

Camera

Android: CameraX PreviewView + ImageCapture. iOS: AVFoundation (scaffold).

Image Picker

Android: ActivityResultContracts.GetContent. iOS: UIImagePicker (scaffold).

ML Inference

Android: TFLite Java API + GPU delegate. iOS: TFLite C API (scaffold).

Permissions

Android: ActivityResultContracts.RequestPermission. iOS: AVFoundation auto-prompt.

Key Features

  • Touch-to-compare — Hold finger on the result to see the original; release to see the enhanced version. Built with Compose's pointerInput + detectTapGestures(onPress).
  • GPU-accelerated inference — On Android, uses GpuDelegate via CompatibilityList to determine device support at runtime, falling back to CPU if unavailable.
  • Zero-copy model loading — The 39 MB TFLite model is loaded via Android's AssetFileDescriptor + FileChannel.map() for direct mmap access without copying into heap.
  • Modular architecture — Clean separation between UI (commonMain), platform services (androidMain/iosMain), and business logic (commonMain/domain).

Building

Prerequisites

  • JDK 17+ (recommended: OpenJDK 21)
  • Android SDK (API 34)
  • Xcode 16+ (iOS only)

Android

./gradlew :composeApp:assembleDebug
adb install -r composeApp/build/outputs/apk/debug/composeApp-debug.apk

iOS Framework

./gradlew :composeApp:linkDebugFrameworkIosSimulatorArm64
open iosApp/iosApp.xcodeproj  # then Cmd+R in Xcode

Dependencies

  • Kotlin 2.0.21 + Compose Multiplatform 1.7.1
  • TensorFlow Lite 2.14.0 + GPU delegate
  • CameraX 1.3.4 (Android)
  • AndroidX Activity Compose + Lifecycle
  • Kotlinx Coroutines 1.9.0
